# 🏗️ Urbanedge Construction

## 📌 About the Project

Urbanedge Construction is a full-stack web application built to showcase construction services and manage content dynamically through an admin panel. The project focuses on modern UI, clean structure, and efficient API integration.

It includes both a **frontend client** built with React and a **backend API** powered by Laravel.

---

## ⚙️ Tech Stack

### Frontend

* React (Vite)
* React Router DOM
* Bootstrap
* React Hook Form
* React Toastify

### Backend

* Laravel (REST API)
* MySQL (Database)

---

## ✨ Features

### 🔹 Admin Panel

* Create, Edit, and Delete Services, Projects, Articles, Members and Testimonials
* Image Upload with Preview
* Form Validation using React Hook Form
* Toast Notifications for user feedback
* Loading states for better UX

### 🔹 Frontend Website

* Dynamic Services, Projects, Articles, Members, and Testimonials Listing
* Detail Pages for Services and Projects and Blogs
* Display of Services, Projects, Articles and Team Members
* Testimonials Showcase Section
* Reusable Components (Header, Footer, Cards)
* API-based dynamic content rendering

---

## 🔄 API Integration

* Fetching data from Laravel backend
* Secure API requests using authentication token
* Handling async operations with proper state management

---

## 🚀 Future Improvements

This project is currently focused on learning and implementing core React concepts such as component structure, API integration, form handling, and state management.

One planned improvement for future iterations is the implementation of a **client-side caching system for API data**.

### 🔹 Why caching?

* To reduce repeated API calls
* To improve performance and load times
* To provide a smoother user experience

### 🔹 Planned approach

* React state (in-memory caching)
* Browser storage like `localStorage`
* Advanced libraries such as React Query (TanStack Query)

### 🔹 Current decision

Caching is intentionally not implemented yet to keep the project focused on fundamentals. This feature will be explored in future projects.
